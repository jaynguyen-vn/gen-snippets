## Architecture Highlights

### Threading Model
- **Reader-Writer Pattern:** DispatchQueue(concurrent) for snippet/category storage
- **Fast Callback Path:** os_unfair_lock for TextReplacementService callback performance
- **Thread-Safe Data Structures:** NSLock in SuffixTree nodes

### Data Persistence
- **Local Storage:** UserDefaults at ~/Library/Preferences/Jay8448.Gen-Snippets.plist
- **Rich Content:** File system at ~/Library/Application Support/GenSnippets/RichContent/
- **Caching Layer:** LocalStorageService maintains in-memory cache with max 1,000 items
- **Batch Operations:** Timer-based debouncing for grouped saves

### Text Replacement Flow
```
User types characters
     ↓
CGEvent tap monitors keyboard (system-wide)
     ↓
TextReplacementService.buffer accumulates (last 50 chars, 15-second timeout)
     ↓
TrieNode suffix tree checks for matches (O(m) where m = command length)
     ↓
Match found → EdgeCaseHandler determines app-specific behavior
     ↓
Delete command via backspaces (with app-specific timing)
     ↓
Replace text insertion (clipboard paste for browsers, key events for standard apps)
     ↓
Dynamic keyword replacement (clipboard, UUID, date, time, etc.)
```

### State Management
```
UserDefaults
     ↓
LocalStorageService (cache + batch saves)
     ↓
LocalSnippetsViewModel & CategoryViewModel (@Published)
     ↓
SwiftUI Views (observe @ObservedObject or @StateObject)
     ↓
TextReplacementService (notified of updates via NotificationCenter)
```
